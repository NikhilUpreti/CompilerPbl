Code Generation

Turning Intermediate Code into Target Machine Code

üìò What is Code Generation?

Code Generation is the final phase of a compiler. It takes the optimized intermediate representation (IR) of a program and translates it into the final machine-level or assembly code, which can be directly executed by the hardware or further assembled and linked.

üéØ Purpose of Code Generation

  • Convert IR to target language (assembly or machine code)
  • Allocate machine resources efficiently (registers, memory)
  • Preserve the semantics of the original program
  • Generate fast and compact code

üß† Input and Output

  • Input: Intermediate Code (e.g., Three-Address Code)
  • Output: Target Code (e.g., Assembly/Machine Code)

‚öôÔ∏è Activities in Code Generation

  • Instruction Selection: Choosing appropriate machine instructions
  • Register Allocation: Assigning variables to machine registers efficiently
  • Instruction Scheduling: Reordering instructions to avoid pipeline stalls and delays
  • Address and Offset Calculation: Computing memory addresses and stack offsets

üìê Example

Intermediate Code:

t1 = a + b
t2 = t1 * c
d = t2

Target Code (x86-like Assembly):

MOV R1, a
ADD R1, b
MUL R1, c
MOV d, R1

üîç Register Allocation

Efficient use of a limited number of CPU registers is essential for fast code. Techniques like graph coloring and live variable analysis are used to minimize memory loads/stores and maximize register reuse.

üß© Code Generator Design Issues

  • Instruction Set of the target machine (RISC vs CISC)
  • Number of available registers
  • Calling conventions and stack layout
  • Performance vs code size trade-offs

üõ†Ô∏è Tools and Technologies

  • LLVM: Open-source compiler framework capable of generating native code
  • GCC: GNU Compiler Collection for various languages and architectures
  • Assembler: Converts assembly code into machine code (e.g., NASM, GAS)
  • JVM/CLR: Platforms that generate bytecode for virtual machines

üöÄ Summary

Code Generation is a crucial stage in compiler construction. It bridges the gap between high-level source code and low-level machine instructions. A good code generator ensures that the final output is not only correct but also efficient and optimized for the specific hardware it targets.
